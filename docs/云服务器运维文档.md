## 一、服务管理

### 1.1 使用启动脚本管理

#### 前提条件

确保已创建并配置好启动脚本：
- `start.sh` - 启动脚本
- `stop.sh` - 停止脚本
- `restart.sh` - 重启脚本

脚本位置通常在：`/opt/test-api/` 或 `/root/test-api/`

#### 启动服务

```bash
# 进入应用目录
cd /opt/test-api

# 启动服务
./start.sh
```

**输出示例：**
```
Starting test-api...
Log directory: /opt/test-api/logs
WECHAT_APPID: wx12345678...
test-api started with PID 12345
Log directory: /opt/test-api/logs
Application log: /opt/test-api/logs/test-api.log
Error log: /opt/test-api/logs/test-api-error.log
Startup log: /opt/test-api/logs/startup.log
```

#### 停止服务

```bash
# 进入应用目录
cd /opt/test-api

# 停止服务
./stop.sh
```

**输出示例：**
```
Stopping test-api (PID: 12345)...
test-api stopped
```

#### 重启服务

```bash
# 进入应用目录
cd /opt/test-api

# 重启服务
./restart.sh
```

**输出示例：**
```
Restarting test-api...
Stopping test-api (PID: 12345)...
test-api stopped
Starting test-api...
test-api started with PID 12346
```

### 1.2 使用 Systemd 管理（推荐）

#### 服务文件位置

Systemd 服务文件通常位于：`/etc/systemd/system/test-api.service`

#### 启动服务

```bash
# 启动服务
sudo systemctl start test-api

# 查看启动状态
sudo systemctl status test-api
```

#### 停止服务

```bash
# 停止服务
sudo systemctl stop test-api

# 查看停止状态
sudo systemctl status test-api
```

#### 重启服务

```bash
# 重启服务
sudo systemctl restart test-api

# 查看重启状态
sudo systemctl status test-api
```

#### 查看服务状态

```bash
# 查看服务详细状态
sudo systemctl status test-api

# 查看服务是否运行
sudo systemctl is-active test-api

# 查看服务是否启用开机自启
sudo systemctl is-enabled test-api
```

#### 设置开机自启

```bash
# 启用开机自启
sudo systemctl enable test-api

# 禁用开机自启
sudo systemctl disable test-api
```

#### 查看服务日志

```bash
# 实时查看服务日志
sudo journalctl -u test-api -f

# 查看最近 100 行日志
sudo journalctl -u test-api -n 100

# 查看今天的日志
sudo journalctl -u test-api --since today

# 查看指定时间段的日志
sudo journalctl -u test-api --since "2024-01-01 00:00:00" --until "2024-01-01 23:59:59"
```

#### 重新加载服务配置

修改服务文件后，需要重新加载：

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 重启服务使配置生效
sudo systemctl restart test-api
```

## 二、日志查看

### 2.1 实时查看日志

#### 查看应用主日志

```bash
# 实时查看应用日志（推荐）
tail -f /opt/test-api/logs/test-api.log

# 或使用 less 查看（支持搜索）
less +F /opt/test-api/logs/test-api.log
# 按 Ctrl+C 退出跟踪模式，输入 /关键词 搜索，按 q 退出
```

#### 查看错误日志

```bash
# 实时查看错误日志
tail -f /opt/test-api/logs/test-api-error.log
```

#### 查看启动日志

```bash
# 查看启动日志
tail -f /opt/test-api/logs/startup.log
```

#### 同时查看多个日志文件

```bash
# 使用 multitail（需要先安装：apt-get install multitail）
multitail /opt/test-api/logs/test-api.log /opt/test-api/logs/test-api-error.log

# 或使用 tail 同时跟踪多个文件
tail -f /opt/test-api/logs/test-api.log /opt/test-api/logs/test-api-error.log
```

---

### 2.2 查看历史日志

#### 查看最近的日志

```bash
# 查看最后 100 行
tail -n 100 /opt/test-api/logs/test-api.log

# 查看最后 50 行错误日志
tail -n 50 /opt/test-api/logs/test-api-error.log

# 查看前 100 行
head -n 100 /opt/test-api/logs/test-api.log
```

#### 搜索日志内容

```bash
# 搜索包含关键字的日志
grep "ERROR" /opt/test-api/logs/test-api.log

# 搜索并显示上下文（前后各 5 行）
grep -C 5 "ERROR" /opt/test-api/logs/test-api.log

# 搜索并高亮显示
grep --color=always "ERROR" /opt/test-api/logs/test-api.log | less -R

# 搜索多个关键字（或关系）
grep -E "ERROR|WARN" /opt/test-api/logs/test-api.log

# 搜索并统计出现次数
grep -c "ERROR" /opt/test-api/logs/test-api.log

# 搜索最近 1 小时的日志（需要日志包含时间戳）
grep "$(date -d '1 hour ago' '+%Y-%m-%d %H')" /opt/test-api/logs/test-api.log
```

#### 查看指定日期的日志

```bash
# 查看指定日期的日志文件
cat /opt/test-api/logs/test-api-2024-01-01.0.log

# 或使用 less 查看（支持搜索和翻页）
less /opt/test-api/logs/test-api-2024-01-01.0.log

# 查看所有日志文件
ls -lh /opt/test-api/logs/
```

#### 统计日志信息

```bash
# 统计错误日志数量
grep -c "ERROR" /opt/test-api/logs/test-api.log

# 统计今天的错误数量
grep "$(date '+%Y-%m-%d')" /opt/test-api/logs/test-api-error.log | wc -l

# 查看日志文件大小
du -sh /opt/test-api/logs/*

# 查看日志文件数量
ls -1 /opt/test-api/logs/*.log | wc -l
```

---

### 2.3 日志文件位置

#### 日志目录结构

```
/opt/test-api/logs/
├── test-api.log                    # 当前应用日志（可能已滚动）
├── test-api-2024-01-01.0.log      # 按日期滚动的日志文件
├── test-api-2024-01-01.1.log      # 同一天多个文件（按大小滚动）
├── test-api-error.log              # 当前错误日志
├── test-api-error-2024-01-01.0.log # 按日期滚动的错误日志
└── startup.log                     # 启动日志
```

#### 日志文件说明

- **test-api.log**: 应用主日志，包含所有级别的日志
- **test-api-error.log**: 只包含 ERROR 级别的日志
- **startup.log**: 应用启动时的控制台输出
- **按日期滚动的文件**: 每天自动生成新文件，格式为 `test-api-YYYY-MM-DD.N.log`

#### 日志保留策略

- **保留时间**: 30 天
- **单个文件大小**: 最大 10MB
- **总大小限制**: 应用日志不超过 1GB，错误日志不超过 500MB
- **自动清理**: 超过保留时间或大小限制的日志会自动删除

## 三、服务状态监控

### 3.1 检查服务运行状态

#### 使用 Systemd 检查

```bash
# 查看服务状态
sudo systemctl status test-api

# 查看服务是否运行
sudo systemctl is-active test-api
# 输出：active 或 inactive

# 查看服务是否启用
sudo systemctl is-enabled test-api
# 输出：enabled 或 disabled
```

#### 使用进程检查

```bash
# 查看进程是否存在
ps aux | grep test-api | grep -v grep

# 查看进程详细信息
ps -ef | grep test-api | grep -v grep

# 查看进程 ID
cat /opt/test-api/test-api.pid
```

#### 使用 PID 文件检查

```bash
# 检查 PID 文件是否存在
ls -l /opt/test-api/test-api.pid

# 读取 PID
cat /opt/test-api/test-api.pid

# 验证 PID 对应的进程是否运行
PID=$(cat /opt/test-api/test-api.pid)
ps -p $PID
```

---

### 3.2 检查端口监听

#### 检查端口是否监听

```bash
# 使用 netstat 检查（需要安装：apt-get install net-tools）
netstat -tlnp | grep 8080

# 使用 ss 检查（推荐，系统自带）
ss -tlnp | grep 8080

# 使用 lsof 检查（需要安装：apt-get install lsof）
lsof -i:8080
```

#### 检查端口占用情况

```bash
# 查看所有监听的端口
ss -tlnp

# 查看特定端口
ss -tlnp | grep :8080

# 查看端口占用进程
fuser 8080/tcp
```

---

### 3.3 检查进程信息

#### 查看进程详细信息

```bash
# 查看进程基本信息
ps aux | grep test-api | grep -v grep

# 查看进程树
pstree -p $(cat /opt/test-api/test-api.pid)

# 查看进程资源使用情况
top -p $(cat /opt/test-api/test-api.pid)

# 或使用 htop（需要安装：apt-get install htop）
htop -p $(cat /opt/test-api/test-api.pid)
```

#### 查看 JVM 信息

```bash
# 查看 JVM 进程信息
jps -lvm | grep test-api

# 查看 JVM 堆内存使用（需要安装 JDK）
jmap -heap $(cat /opt/test-api/test-api.pid)

# 查看 JVM 线程信息
jstack $(cat /opt/test-api/test-api.pid)
```

## 四、常见运维操作

### 4.1 服务启动

#### 启动前检查

```bash
# 1. 检查 JAR 文件是否存在
ls -lh /opt/test-api/test-api-0.0.1-SNAPSHOT.jar

# 2. 检查环境变量是否设置
echo $WECHAT_APPID
echo $WECHAT_SECRET

# 3. 检查日志目录权限
ls -ld /opt/test-api/logs

# 4. 检查端口是否被占用
ss -tlnp | grep 8080

# 5. 检查数据库连接（可选）
mysql -u root -p -e "SELECT 1"
```

#### 启动服务

```bash
# 方式1：使用启动脚本
cd /opt/test-api
./start.sh

# 方式2：使用 Systemd
sudo systemctl start test-api

# 方式3：手动启动（用于调试）
cd /opt/test-api
export WECHAT_APPID=your_appid
export WECHAT_SECRET=your_secret
nohup java -Xms512m -Xmx1024m -XX:+UseG1GC \
    -jar test-api-0.0.1-SNAPSHOT.jar \
    --spring.profiles.active=prod \
    > logs/startup.log 2>&1 &
```

#### 启动后验证

```bash
# 1. 检查进程是否启动
ps aux | grep test-api | grep -v grep

# 2. 检查端口是否监听
ss -tlnp | grep 8080

# 3. 查看启动日志
tail -f /opt/test-api/logs/startup.log

# 4. 查看应用日志
tail -f /opt/test-api/logs/test-api.log

# 5. 测试接口（可选）
curl http://localhost:8080/api/game/user-data?userId=123456
```

---

### 4.2 服务停止

#### 优雅停止

```bash
# 方式1：使用停止脚本
cd /opt/test-api
./stop.sh

# 方式2：使用 Systemd
sudo systemctl stop test-api

# 方式3：使用 kill 命令（发送 SIGTERM 信号）
kill $(cat /opt/test-api/test-api.pid)

# 方式4：使用 killall
killall -15 java
```

#### 强制停止

```bash
# 如果优雅停止失败，使用强制停止
kill -9 $(cat /opt/test-api/test-api.pid)

# 或使用 killall
killall -9 java
```

#### 停止后验证

```bash
# 1. 检查进程是否已停止
ps aux | grep test-api | grep -v grep

# 2. 检查端口是否已释放
ss -tlnp | grep 8080

# 3. 检查 PID 文件是否已删除
ls -l /opt/test-api/test-api.pid
```

### 4.5 更新应用版本

#### 更新步骤

```bash
# 1. 备份当前版本
cd /opt/test-api
cp test-api-0.0.1-SNAPSHOT.jar test-api-0.0.1-SNAPSHOT.jar.backup

# 2. 停止服务
./stop.sh
# 或
sudo systemctl stop test-api

# 3. 上传新版本 JAR 文件（使用 scp 或其他方式）
# scp target/test-api-0.0.1-SNAPSHOT.jar user@server:/opt/test-api/

# 4. 验证新 JAR 文件
ls -lh test-api-0.0.1-SNAPSHOT.jar

# 5. 启动服务
./start.sh
# 或
sudo systemctl start test-api

# 6. 验证服务是否正常
tail -f logs/test-api.log
curl http://localhost:8080/api/game/user-data?userId=123456
```

#### 回滚操作

如果新版本有问题，可以快速回滚：

```bash
# 1. 停止服务
./stop.sh

# 2. 恢复备份版本
mv test-api-0.0.1-SNAPSHOT.jar.backup test-api-0.0.1-SNAPSHOT.jar

# 3. 启动服务
./start.sh
```

## 五、故障排查

### 5.1 服务无法启动

#### 检查清单

```bash
# 1. 检查 JAR 文件是否存在且可读
ls -lh /opt/test-api/test-api-0.0.1-SNAPSHOT.jar

# 2. 检查 Java 是否安装
java -version

# 3. 检查环境变量是否设置
echo $WECHAT_APPID
echo $WECHAT_SECRET

# 4. 检查日志目录权限
ls -ld /opt/test-api/logs
touch /opt/test-api/logs/test.log && rm /opt/test-api/logs/test.log

# 5. 检查端口是否被占用
ss -tlnp | grep 8080

# 6. 查看启动日志
cat /opt/test-api/logs/startup.log

# 7. 手动启动查看错误信息
cd /opt/test-api
java -jar test-api-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

#### 常见错误及解决方案

**错误1：JAR 文件不存在**
```bash
# 解决：上传 JAR 文件到正确位置
scp target/test-api-0.0.1-SNAPSHOT.jar user@server:/opt/test-api/
```

**错误2：环境变量未设置**
```bash
# 解决：设置环境变量（参考 Ubuntu 环境变量设置指南）
export WECHAT_APPID=your_appid
export WECHAT_SECRET=your_secret
```

**错误3：端口被占用**
```bash
# 解决：查找并杀死占用端口的进程
lsof -i:8080
kill -9 <PID>
```

**错误4：数据库连接失败**
```bash
# 解决：检查数据库服务是否运行
systemctl status mysql
# 检查数据库配置
cat /opt/test-api/config/application-prod.properties | grep datasource
```

### 5.2 服务异常退出

#### 检查步骤

```bash
# 1. 查看错误日志
tail -100 /opt/test-api/logs/test-api-error.log

# 2. 查看应用日志
tail -100 /opt/test-api/logs/test-api.log

# 3. 查看系统日志
sudo journalctl -u test-api -n 100

# 4. 检查系统资源
free -h
df -h
top

# 5. 检查是否有 OOM（内存溢出）
dmesg | grep -i "out of memory"
grep -i "outofmemory" /opt/test-api/logs/test-api-error.log
```

#### 常见原因

1. **内存不足（OOM）**
   - 解决：增加 JVM 内存或优化应用
2. **数据库连接超时**
   - 解决：检查数据库服务，调整连接池配置
3. **未捕获的异常**
   - 解决：查看错误日志，修复代码问题
4. **磁盘空间不足**
   - 解决：清理日志文件，释放磁盘空间

### 5.3 端口被占用

#### 查找占用端口的进程

```bash
# 方法1：使用 lsof
lsof -i:8080

# 方法2：使用 netstat
netstat -tlnp | grep 8080

# 方法3：使用 ss
ss -tlnp | grep 8080

# 方法4：使用 fuser
fuser 8080/tcp
```

#### 释放端口

```bash
# 方法1：杀死占用端口的进程
kill -9 <PID>

# 方法2：使用 fuser 直接杀死
fuser -k 8080/tcp
```

---

### 5.4 内存不足

#### 检查内存使用

```bash
# 查看系统内存
free -h

# 查看进程内存使用
ps aux --sort=-%mem | head -10

# 查看 JVM 堆内存
jmap -heap $(cat /opt/test-api/test-api.pid)
```

#### 解决方案

```bash
# 1. 调整 JVM 内存参数（修改启动脚本）
# 将 -Xmx1024m 改为 -Xmx512m（根据服务器内存调整）

# 2. 重启服务
./restart.sh

# 3. 如果服务器内存不足，考虑升级服务器配置
```

---

### 5.5 数据库连接失败

#### 检查步骤

```bash
# 1. 检查数据库服务是否运行
systemctl status mysql

# 2. 测试数据库连接
mysql -u root -p -e "SELECT 1"

# 3. 检查数据库配置
cat /opt/test-api/config/application-prod.properties | grep datasource

# 4. 检查防火墙
sudo ufw status
# 或
sudo firewall-cmd --list-all

# 5. 查看应用日志中的数据库错误
grep -i "datasource\|database\|mysql" /opt/test-api/logs/test-api-error.log
```

#### 解决方案

```bash
# 1. 启动数据库服务
sudo systemctl start mysql

# 2. 检查数据库用户权限
mysql -u root -p
GRANT ALL PRIVILEGES ON test.* TO 'testuser'@'localhost';
FLUSH PRIVILEGES;

# 3. 检查防火墙规则
sudo ufw allow 3306/tcp
```

## 七、备份与恢复

### 7.1 日志备份

#### 备份日志文件

```bash
# 创建备份目录
mkdir -p /backup/test-api/logs

# 备份日志文件
tar -czf /backup/test-api/logs/logs-$(date +%Y%m%d).tar.gz /opt/test-api/logs/

# 或使用 rsync
rsync -av /opt/test-api/logs/ /backup/test-api/logs/
```

#### 清理旧日志

```bash
# 删除 30 天前的日志文件
find /opt/test-api/logs/ -name "*.log" -mtime +30 -delete

# 或只保留最近 7 天的日志
find /opt/test-api/logs/ -name "test-api-*.log" -mtime +7 -delete
```

### 7.2 配置文件备份

#### 备份配置文件

```bash
# 创建备份目录
mkdir -p /backup/test-api/config

# 备份配置文件
cp /opt/test-api/config/application-prod.properties /backup/test-api/config/application-prod.properties.$(date +%Y%m%d)

# 备份启动脚本
cp /opt/test-api/start.sh /backup/test-api/start.sh.$(date +%Y%m%d)
cp /opt/test-api/stop.sh /backup/test-api/stop.sh.$(date +%Y%m%d)
```

### 7.3 数据库备份

#### 备份数据库

